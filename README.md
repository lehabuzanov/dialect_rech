# Диалектная речь: Streamlit-приложение для орфографии и транскрипции

## Что это такое

Это локальное Streamlit-приложение для распознавания русской диалектной звучащей речи из загружаемого аудиофайла.  
Приложение:

- принимает аудиофайлы `wav`, `mp3`, `flac`, `ogg`, `m4a`;
- показывает сведения о файле и простую визуализацию звуковой волны;
- распознаёт речь локально в Python через `faster-whisper`;
- жёстко фиксирует язык распознавания как русский (`ru`);
- работает только в режиме `transcribe`, без перевода;
- выводит орфографическую запись;
- строит отдельную русскую практическую транскрипцию в кириллице;
- позволяет вручную редактировать оба результата;
- умеет озвучивать текст локально через Silero TTS;
- даёт скачать результаты в `.txt` и `.json`.

## Архитектура

Проект разделён на простые модули:

- `app.py`  
  Streamlit-интерфейс, состояние сессии, кнопки, плееры, скачивание результатов.
- `src/asr.py`  
  Работа с `faster-whisper`, выбор модели, загрузка и кэширование модели.
- `src/audio_utils.py`  
  Проверка файла, чтение метаданных, декодирование аудио, подготовка волны, временные файлы.
- `src/transcription.py`  
  Нормализация орфографического текста и rule-based практическая транскрипция в кириллице.
- `src/tts.py`  
  Локальный русский TTS через Silero.
- `src/ui_helpers.py`  
  Инициализация `st.session_state`, статусная область, экспорт результатов.

## Технологический стек

- Интерфейс: `Streamlit`
- ASR backend: `faster-whisper`
- Основная модель по умолчанию: `openai/whisper-large-v3-turbo`
- Озвучивание текста: `Silero TTS`
- График аудио: `Plotly`
- Декодирование аудио без системного `ffmpeg`: `PyAV`

## Как работает распознавание

### Основная модель

По умолчанию выбрана модель:

`openai/whisper-large-v3-turbo`

Так как `faster-whisper` работает с моделями в формате CTranslate2, приложение при первом запуске:

1. скачивает исходную модель `openai/whisper-large-v3-turbo`;
2. локально конвертирует её в формат CTranslate2;
3. сохраняет результат в пользовательский кэш;
4. при следующих запусках использует уже готовую локальную копию.

В интерфейсе также есть реальные fallback-модели:

- `Systran/faster-whisper-large-v3`
- `Systran/faster-whisper-medium`
- `Systran/faster-whisper-small`

Это не декоративный выбор: при нажатии кнопки `Распознать` используется именно выбранная модель.

### Язык и режим

- язык принудительно фиксируется как `ru`;
- используется только задача `transcribe`;
- перевод отключён;
- включён `vad_filter`, чтобы аккуратнее обрабатывать паузы;
- для длинных файлов используется batched inference pipeline.

## Как получается орфографическая запись

После распознавания приложение:

1. собирает все сегменты;
2. очищает лишние пробелы;
3. нормализует базовую пунктуацию;
4. выводит результат в большое редактируемое поле `Орфографическая запись`.

Пользователь может вручную исправить текст прямо в интерфейсе.

## Как получается транскрипция

Поле `Транскрипция` строится отдельно от орфографии.  
Сейчас используется модульная rule-based практическая транскрипция в кириллице:

- без IPA;
- без латиницы;
- без внешних API;
- с отдельным модулем, который потом можно заменить на более продвинутый.

Примерно учитываются:

- некоторые частотные разговорные преобразования;
- упрощение сочетаний вроде `тся -> ца`;
- отдельные случаи оглушения на конце слова;
- приведение результата к русской практической форме записи.

Пользователь также может вручную отредактировать транскрипцию.

## Как работает озвучивание текста

Озвучивание выполняется локально через Silero TTS.

Доступны кнопки:

- `Прослушать орфографию`
- `Прослушать транскрипцию`

После синтеза приложение показывает встроенный аудиоплеер Streamlit.  
Перед озвучиванием транскрипционный текст мягко нормализуется до произносимой русской формы.

## Установка

Рекомендуется Python `3.11`.

### 1. Клонировать репозиторий

```bash
git clone https://github.com/lehabuzanov/dialect_rech.git
cd dialect_rech
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

Linux/macOS:

```bash
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

## Запуск приложения

```bash
streamlit run app.py
```

После запуска откроется локальный веб-интерфейс Streamlit.

## Как пользоваться

1. Загрузите аудиофайл.
2. Проверьте информацию о файле и прослушайте исходное аудио.
3. Посмотрите на график волны для визуального контроля.
4. При необходимости выберите другую модель в sidebar.
5. Нажмите `Распознать`.
6. Отредактируйте `Орфографическую запись`, если нужно.
7. Нажмите `Обновить транскрипцию из орфографии`, если меняли орфографию.
8. Прослушайте орфографию или транскрипцию.
9. Скачайте результаты.

## Где хранятся модели

Большие веса не кладутся в Git-репозиторий.

Модели и конвертированные веса сохраняются локально в кэше пользователя, например:

```text
~/.cache/dialect_rech/models/
```

На Windows это будет путь внутри профиля пользователя.

## Ограничения

- приложение не использует микрофон;
- не использует облачные speech-to-text API;
- не использует внешние сервисы перевода;
- качество на очень шумной записи или при крайне сильной диалектности всё равно зависит от качества исходного аудио;
- первая загрузка основной модели может занять заметное время;
- для слабого CPU модель `openai/whisper-large-v3-turbo` может работать медленно;
- текущая транскрипция практическая и rule-based, а не полноценная фонологическая экспертная разметка.

## Возможные проблемы и решения

### 1. Модель долго загружается при первом запуске

Это ожидаемо.  
При первом старте приложение скачивает и подготавливает модель локально.

Что делать:

- дождаться завершения первого запуска;
- проверить доступ в интернет для первичной загрузки модели;
- после первого запуска повторное открытие будет быстрее.

### 2. Ошибка при конвертации `openai/whisper-large-v3-turbo`

Возможные причины:

- нестабильный интернет при первичной загрузке;
- нехватка места на диске;
- несовместимая версия зависимостей.

Что делать:

- повторить запуск;
- проверить свободное место;
- удалить локальный кэш модели и запустить снова.

### 3. Медленно работает на слабом ПК

Что делать:

- выбрать `Systran/faster-whisper-medium`;
- выбрать `Systran/faster-whisper-small`;
- по возможности использовать GPU.

### 4. Ошибка CUDA или GPU

Если GPU-библиотеки недоступны, приложение автоматически пытается работать на CPU с `int8`.

### 5. Silero TTS не синтезирует речь

Что делать:

- проверить, что все зависимости установлены корректно;
- убедиться, что модель TTS смогла один раз скачаться локально;
- попробовать озвучивание на коротком тексте.

### 6. Файл не читается

Что делать:

- проверить, что формат входит в список поддерживаемых;
- попробовать пересохранить файл;
- убедиться, что файл не повреждён.

## Деплой в Streamlit Community Cloud

При создании приложения в Streamlit Community Cloud:

- репозиторий: `lehabuzanov/dialect_rech`
- branch: `main`
- main file path: `app.py`

Если в настройках доступен выбор версии Python, для максимальной совместимости лучше выбрать `Python 3.11` или `Python 3.12`.  
При этом сам проект старается оставаться совместимым и с более новыми окружениями без жёсткой привязки к одной версии Python.

Если Streamlit Cloud запускает приложение на очень новой версии Python, критично важно иметь свежие бинарные колёса для аудио-зависимостей.  
Поэтому в проекте используются более новые диапазоны `numpy` и `av`, чтобы не провоцировать сборку из исходников на стороне Cloud.
